## 本队贡献

### 任务二：RK3588 NPU 上的 ACT 部署

1. 系统对比 full-INT8 / fp16 / "视觉骨干 INT8 + transformer FP16" 混合量化的判向精度；
   发现并解释「模拟器判无损的混合量化，在真 NPU 上反而最差」的反转现象（轮速差信号 ~0.005 与真硬件量化噪声同量级），
   据此定型 fp16 部署。
2. 板上推理程序（`board/`）：用 Rust + RKNN 运行时 C API 实现 ACT 推理，自做预处理对齐（与模拟器输入 corr 0.99997），交叉编译到 aarch64。
3. StarryOS 自烧整盘镜像与上板集成（`starryos_npu/`）——本作品工程贡献的主体，全部为本队原创集成：
   - 从开发板提取 RK3588 引导块，拼成自包含、可 `dd` 烧录的整盘镜像（区别于官方 ostool 串口引导）；
   - 定位并解决一系列跨引导固件、OS、文件系统、应用的问题：真 rootfs 当根分区（非 ramdisk）、置 `EXT4_ERROR_FS` 标志逼只读挂载以绕开该平台 SD 写死锁、删 dtb `/chosen/bootargs` 使 `console=ttyS2` 不被板载 rockchip U-Boot 改写从而绑定串口；
   - 最终在 StarryOS + RK3588 上跑通 ACT 推理，结果与在 Ubuntu 22.04 上实测一致。
   - 完整定位过程见 `starryos_npu/上板调试历程.md`，结果见 `starryos_npu/RESULT_starryos_board.md`。

> 以上 3 与基础版本 tgoskits 的 demo 在目标、形态与所解决的问题上均不同，属本队原创贡献。
