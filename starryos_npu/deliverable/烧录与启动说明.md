# StarryOS + ACT NPU 推理 —— 上板说明（Orange Pi 5 Plus）

本包是在服务器上自建的、带 rknpu 驱动的 StarryOS 镜像 + 我们的 ACT 推理程序。
目标：在 StarryOS 上用真 NPU 跑 ACT 推理，串口看判向输出。

## 包内文件
- `starryos-orangepi5plus-rknpu.bin` —— StarryOS 内核 Image（含 rknpu 驱动，ARM64 booti 可引导）
- `orangepi-5-plus.dtb` —— 设备树（含 NPU 节点）
- `rootfs-aarch64-act.img` —— 根文件系统（已含 glibc 运行时 + /act_rknn 我们的程序+模型+帧）
- `orangepi-5-plus-uboot.toml` —— U-Boot/串口配置参考

## 串口
USB-TTL 接板子调试串口，**波特率 1500000**，8N1。

---

## 方式 A（推荐，tgoskits 标准，无需烧整盘卡）：ostool U-Boot 引导

这是 tgoskits 跑 Orange Pi 5 Plus NPU demo 的标准方式：通过 U-Boot 把内核+rootfs 载入板子内存启动。
在“接了板子的电脑”上（已 clone tgoskits、装好 nightly 工具链）：

```bash
# 把本包内核/dtb/rootfs 放到对应位置后：
cargo xtask starry uboot -b OrangePi-5-Plus
# 或参考 tgoskits 文档的 cargo starry uboot 流程，指定本包的 dtb 与 rootfs
```
ostool 会复位板子、进 U-Boot、加载内核+rootfs 并 `booti` 启动。

## 方式 B：手动 U-Boot 从 TF 卡引导

1. 把 `starryos-orangepi5plus-rknpu.bin`、`orangepi-5-plus.dtb`、`rootfs-aarch64-act.img` 拷到一张 TF 卡的某个分区（如 FAT 的 boot 分区）。
2. 板子上电，串口里按键打断进 U-Boot。
3. 在 U-Boot 里加载并启动（地址按板子内存调整）：
   ```
   # 从 mmc 读入内核 / dtb / rootfs(ramdisk) 到内存，然后 booti
   load mmc 1:1 0x02080000 starryos-orangepi5plus-rknpu.bin
   load mmc 1:1 0x0a100000 orangepi-5-plus.dtb
   load mmc 1:1 0x0b000000 rootfs-aarch64-act.img
   booti 0x02080000 0x0b000000:<rootfs字节数> 0x0a100000
   ```

> 注：rootfs 投递（ramdisk 入内存 vs 从分区挂载）的细节，第一次上板可能需按串口报错微调；
> 方式 A 的 ostool 会自动处理这些，故优先用 A。

## 期望输出（成功）

串口里应打印每帧：
```
frame_xxxxxx.jpg: left_vel=.. right_vel=.. diff=±.. decision=left/right (xx ms)
...
summary: 56/56 frames, avg inference .. ms
```
拿 `/act_rknn/EXPECTED.csv`（gt 参考方向）肉眼对照 decision。手动跑：`sh /act_rknn/init.sh`。

## 已在服务器上验证（无需板子）
- 内核含 rknpu 驱动、设备树含 NPU 节点（电源/时钟/中断）。
- 用 qemu-user 验通用户态链路：act-rknn 用注入的 glibc + librknnrt 成功加载、跑到 rknn_init，
  仅因 x86 无 NPU 设备而停（板上由 rknpu 驱动提供该设备）。
