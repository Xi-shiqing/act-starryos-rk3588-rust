# 如何重建可烧录镜像

镜像的 release：[https://github.com/Xi-shiqing/act-starryos-rk3588-rust/releases/tag/v1.0.0](https://github.com/Xi-shiqing/act-starryos-rk3588-rust/releases/tag/v1.0.0)

`app/build_sd_image.sh` 只负责最后一步组装：把引导块 + 内核 + dtb + rootfs 拼成一张可 `dd` 的整盘 TF 卡镜像。这是在所有上游产物已经准备好的情况下执行的。本仓库默认不收录这些大产物（见 `.gitignore`），因此从一个干净的 clone 直接跑脚本会因缺少输入而退出（脚本带前置检查，会明确提示缺哪个、去哪拿）。

## 依赖关系

```
tgoskits 编内核 ─┐
                 ├─► deliverable/starryos-orangepi5plus-rknpu.bin   (内核 Image, 14MB)
                 └─► deliverable/orangepi-5-plus.dtb                (设备树, 274KB)

官方 musl rootfs + 注入 ──► app/inject_rootfs.sh ──► deliverable/rootfs-aarch64-act.img  (1GB)
   (aarch64 glibc 运行时 + /act_rknn 推理载荷)

从开发板抠出 ────────────► bootloader/bootblob.bin(.gz)             (RK3588 idbloader+u-boot.itb, 16MB)

以上四者 ──► app/build_sd_image.sh ──► sdcard/starryos-act-orangepi5plus-sd.img(.gz)
```

## 仓库里已有与没有的

| 输入 | 仓库内 | 如何获得 |
|---|---|---|
| `bootloader/bootblob.bin.gz` | 已有 | 脚本会自动 `gzip -dk` 解压成 `.bin` |
| `app/orangepi-5-plus.nobootargs.dtb` | 已有 | 由 `deliverable/orangepi-5-plus.dtb` 删 `/chosen/bootargs` 得到（脚本内置该步骤） |
| `deliverable/starryos-orangepi5plus-rknpu.bin` | 没有 | tgoskits 编内核：`cargo starry defconfig orangepi-5-plus`（启用 `rknpu`）+ `cargo starry build` |
| `deliverable/orangepi-5-plus.dtb` | 没有 | 随上面内核产出 |
| `deliverable/rootfs-aarch64-act.img` | 没有 | `app/inject_rootfs.sh`（官方 musl rootfs 注入 aarch64 glibc 运行时 + `/act_rknn` 载荷） |

> 这些产物的来历与脚本，详见 `deliverable/README.md` 与 `app/`（`inject_rootfs.sh`、`assemble_payload.sh`、`verify_qemu.sh`）。

## 已有上游产物时用一条命令组装

把上表后三项放到 `deliverable/`，然后：

```bash
sudo bash app/build_sd_image.sh
```

> 改测试帧数（如 56 → 完整 666）：帧装在 rootfs 的 `/act_rknn/frames` 里，`build_sd_image.sh` 只做组装、不碰帧。
> 要换帧必须先把 `app/payload/act_rknn/frames` 备好对应帧（完整 666 帧即 `data/frames` 全量），再依次跑：
> ```bash
> sudo bash app/inject_rootfs.sh deliverable/rootfs-aarch64-act.img   # 重新注入 666 帧载荷
> sudo bash app/build_sd_image.sh                                     # 再组装镜像
> ```
> 本仓库当前 payload 已是完整 666 帧，镜像内 `/act_rknn/frames` 即 666 帧。

脚本会自动：前置检查 → 解压引导块 → 收缩 rootfs、对齐 ext4 特性、置只读标志 → 删 dtb 的 `/chosen/bootargs` → 写 extlinux → 组装 GPT 整盘 → 产出并校验：

```
sdcard/starryos-act-orangepi5plus-sd.img        (360MB)
sdcard/starryos-act-orangepi5plus-sd.img.gz     (~185MB，666 帧)
sdcard/SHA256SUMS
```
